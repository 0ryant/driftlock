# Driftlock TDD Skill

## Purpose

Preserve contract-first delivery.

## Sequence

1. Write or update schema.
2. Add a golden fixture.
3. Add a failing test.
4. Implement smallest behavior.
5. Run contract and integration tests.
6. Update task ledger.

## Refuse shortcuts

Do not implement behavior first and backfill tests later unless the task explicitly records an emergency exception.
