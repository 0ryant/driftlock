What happened

One work order. Two diffs. A path-level boundary.

Synthetic ADR-0007 decides to add a Redis-backed session store. Its derived work order adr-0007:T01 authorises writes to exactly crates/session-store/** and tests/session/**, and explicitly names billing (crates/billing/**) and CI/release config as non-goals. The verdict

Compliant passes. Escaping is refused.

check-diff is a reporter — it prints the DiffReport and always exits 0; the verdict is the allowed boolean. The enforcing gate is driftlock complete, which calls the same verify_changed_files and bails with exit 1 when allowed is false. A task can never be marked complete on a diff that escaped its write-set.

What this pack does not prove.

Path-level only — a bounded claim

Sub-file scoping. driftlock checks which files changed, not which lines or symbols. An in-scope file edited beyond intent still passes.

Diff-source trust. The verdict is only as honest as the supplied diff / git diff. A diff that omits a file hides it from the check; driftlock does not independently re-derive the working tree here.

Semantic intent. A file inside the write-set that implements the wrong thing is still allowed: true. driftlock bounds blast radius; it does not judge correctness.

Glob fidelity. Authorisation is glob-based — an over-broad write-set (e.g. crates/**) would legitimately admit billing. The gate is only as strong as the ADR-derived write-set upstream of it.
